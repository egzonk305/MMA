Aufgabe 3.1: Fehlerbehandlung (ca. 20min)

Wenn Sie es nicht schon während der letzten Aufgaben gemacht haben:
Fügen Sie grundlegende Fehlerbehandlung zu Ihren Endpunkten hinzu. Eine gute API gibt aussagekräftige Fehlermeldungen mit passenden HTTP-Statuscodes zurück.

404 (Not Found): Wenn eine Ressource nicht existiert:

app.get("/api/recipes/:id", (req, res) => { const id = Number(req.params.id); const recipe = recipes.find(r => r.id === id); if (!recipe) { res.status(404).json({ error: "Rezept nicht gefunden" }); return; } res.json(recipe); });

400 (Bad Request): Wenn die Eingabe ungültig ist:

app.post("/api/recipes", (req, res) => { const { title, cuisine, prepTime } = req.body; if (!title || typeof title !== "string") { res.status(400).json({ error: "title ist erforderlich und muss ein String sein" }); return; } // ... Rest der Logik });

Implementieren Sie zumindest Fehlerbehandlung für folgende Fälle:

  • Ein Eintrag wurde nicht gefunden → 404 Not Found
  • Die Daten passen nicht zum erwarteten Typ → 400 Bad Request
  • Ein erforderliches Feld fehlt → 400 Bad Request

Aufgabe 3.2: Typdefinitionen & (optional) Destructuring (ca. 30min)

Erstellen Sie eine zentrale Typdefinition und nutzen Sie Destructuring in Ihren Express-Handlern.

Schritt 1: Erstellen Sie eine Datei src/types.ts mit allen Ihren Datentypen:

// src/types.ts export type Recipe = { id: number; title: string; cuisine: string; prepTime: number; };

Schritt 2: Importieren Sie den Typ in server.ts und verwenden Sie ihn überall:

import { Recipe } from "./types"; const recipes: Recipe[] = [ { id: 1, title: "Spaghetti Carbonara", cuisine: "Italienisch", prepTime: 25 }, // ... ];

Schritt 3: Nutzen Sie Object Destructuring in Ihren Express-Handlern, um den Code lesbarer zu machen:

// Vorher: app.post("/api/recipes", (req, res) => { const title = req.body.title; const cuisine = req.body.cuisine; // ... }); // Nachher — mit Destructuring: app.post("/api/recipes", (req, res) => { const { title, cuisine, prepTime } = req.body; // Object Destructuring // ... });

Schritt 4: Probieren Sie auch Array Destructuring aus:

// Die ersten und restlichen Elemente trennen: const [first, ...rest] = recipes; console.log(first); // Erstes Rezept console.log(rest.length); // Anzahl der übrigen

Hinweis: Destructuring mit Defaultwerten ist besonders nützlich für optionale Parameter:

const { name, age = 25 } = user; // age ist 25, falls user.age undefined ist

Verwenden Sie TypeScript-Typen durchgehend — vermeiden Sie any!

Aufgabe 3.3: Code Review mit Kilo Code (ca. 25min)

Erleben Sie zum ersten Mal systematisches Code Review! Kilo Code hat einen speziellen Review Mode, der Ihren Code analysiert und Verbesserungsvorschläge macht.

Schritt 1: Öffnen Sie Kilo Code in VS Code und wechseln Sie in den Review Mode.

Schritt 2: Lassen Sie Ihren gesamten Code reviewen. Kilo Code wird typischerweise Folgendes finden:

  • Fehlende Fehlerbehandlung
  • Inkonsistente Benennung
  • Verwendung von any (sollte vermieden werden!)
  • Sicherheitsprobleme
  • Verbesserungsmöglichkeiten bei der Typisierung

Schritt 3: Beheben Sie mindestens 3 Issues, die das Review identifiziert.
Es sei denn, sie haben bereits optimalen Code erstellt ;-)

Schritt 4: Dokumentieren Sie kurz (in einer Datei REVIEW.md oder als Kommentare), was Sie gelernt haben. Beispiel:

// Vorher (Review-Feedback: "any" vermeiden): app.post("/api/recipes", (req: any, res: any) => { ... }); // Nachher (typisiert): app.post("/api/recipes", (req: Request, res: Response) => { ... });

Tipp: Code Review ist auch für erfahrene Entwickler unverzichtbar! Es geht nicht darum, fehlerfrei zu sein, sondern darum, voneinander zu lernen und die Codequalität zu verbessern.

Optionale Aufgaben (Take-Home Improvements)

Die folgenden Aufgaben sind optional und müssen nicht abgegeben werden.

Aufgabe 3.4: Tailwind Responsiveness (ca. 20min)

Schritt 4: Machen Sie Ihr Layout responsive. Tailwind nutzt Breakpoint-Prefixes:

  • sm: — ab 640px Breite
  • md: — ab 768px Breite
  • lg: — ab 1024px Breite

Beispiel: class="grid grid-cols-1 md:grid-cols-2 lg:grid-cols-3" zeigt 1 Spalte auf Mobilgeräten, 2 auf Tablets und 3 auf Desktop.

Tipp: Die Tailwind-Dokumentation ist hervorragend — nutzen Sie die Suchfunktion, um passende Klassen zu finden!

Aufgabe 3.5: Pagination implementieren (optional)

Fügen Sie Seitennavigation (Pagination) zu Ihrem Listen-Endpunkt hinzu:

GET /api/recipes?page=1&limit=10

Der Endpunkt sollte die Ergebnisse aufteilen und Metadaten zurückgeben:

{ "data": [...], "page": 1, "limit": 10, "total": 42, "totalPages": 5 }

Aufgabe 3.6: Sortierfunktion (optional)

Fügen Sie Sortierungsmöglichkeiten über Query-Parameter hinzu:

GET /api/recipes?sort=title&order=asc GET /api/recipes?sort=prepTime&order=desc

Implementieren Sie die Sortierung im Backend und fügen Sie Sortier-Buttons im Frontend hinzu.
